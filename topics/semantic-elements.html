Overview

HTML5 introduced semantic elements to improve the structure and accessibility of web content. These elements provide meaning to the document structure, aiding both developers and search engines.

1. <header>

The <header> element is used to group introductory content or navigation links.

Syntax:

<header>
    <h1>Page Title</h1>
    <nav>
      <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
      </ul>
    </nav>
  </header>

Example:

Rendered Output:

Welcome to My Website

  • Home
  • About
  • Contact

2. <footer>

The <footer> element contains footer information, such as copyright and links.

Syntax:

<footer>
    <p>Copyright 2024</p>
  </footer>

Example:

Rendered Output:

Copyright 2024

3. <article>

The <article> element represents self-contained content that can be distributed independently.

Syntax:

<article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>

Example:

Rendered Output:

Understanding Semantic HTML

Semantic HTML enhances accessibility and SEO.

4. <section>

The <section> element defines a thematic grouping of content, usually with a heading.

Syntax:

<section>
    <h2>Section Title</h2>
    <p>Section content...</p>
  </section>

Example:

Rendered Output:

About Us

We are dedicated to providing excellent services.

5. <aside>

The <aside> element represents content tangentially related to the main content, such as sidebars.

Syntax:

<aside>
    <p>Related links or information...</p>
  </aside>

Example:

Rendered Output:

Did you know? HTML5 simplifies web development.

6. <figure> & <figcaption>

The <figure> element is used for images or diagrams, and <figcaption> provides a caption.

Syntax:

<figure>
    <img src="image.jpg" alt="Description">
    <figcaption>Caption text</figcaption>
  </figure>
